# AVALON 微信小程序

基于微信小程序的阿瓦隆(Avalon)桌游平台，支持多人实时在线游戏。

## 功能特性

- **5-12人游戏**：支持5-12人同时在线游戏，适配不同人数配置
- **座位号系统**：玩家选择唯一座位号(1-12号玩家)，与线下座位绑定
- **自定义板子**：房主可配置角色组合，支持多种游戏模式
- **灵活开局**：首车头、发车顺序均可指定
- **实时同步**：基于 Express + Socket.io 的玩家状态实时同步
- **完整角色**：梅林、派西维尔、忠臣、莫德雷德、刺客等经典角色
- **投票机制**：队伍投票、任务投票完整实现
- **比赛回顾**：游戏结束后可回顾完整比赛过程

## 前端结构 (miniprogram/)

微信小程序原生开发，使用 WeUI 组件库：

```
miniprogram/
├── app.js              # 小程序入口，生成用户openId
├── app.json            # 小程序配置（页面、窗口样式）
├── app.wxss            # 全局样式
├── project.config.json # 微信开发者工具配置
├── services/
│   └── api.js          # API服务层，指向 https://haoyu-wang141.top:8082
├── pages/              # 页面目录
│   ├── index/          # 首页（创建/加入房间）
│   ├── room/           # 房间页（玩家列表、座位选择）
│   └── game/           # 游戏页（游戏逻辑）
├── utils/              # 工具函数
│   ├── constants.js    # 游戏常量（角色、阶段、配置）
│   └── gameLogic.js    # 游戏逻辑工具
└── images/             # 静态资源
```

## 后端结构 (server/)

Express + Socket.io 后端服务，支持实时通信：

```
server/
├── index.js            # 主入口，Express + Socket.io 服务器
├── package.json        # Node.js 依赖配置
└── routes/             # API路由
    ├── rooms.js        # 房间管理API（创建/加入/离开/准备）
    ├── games.js        # 游戏逻辑API（开始/投票/任务）
    └── messages.js     # 聊天消息API
```

**API端点：**
- `POST /api/rooms/create` - 创建房间
- `POST /api/rooms/join` - 加入房间
- `GET /api/games/:roomId` - 获取游戏状态
- `POST /api/games/start` - 开始游戏
- `POST /api/games/castVote` - 投票
- `POST /api/messages/send` - 发送消息
- `GET /api/health` - 健康检查

**Socket.io 事件：**
- `joinRoom` / `leaveRoom` - 加入/离开房间
- `roomUpdate` / `gameUpdate` - 状态更新广播
- `message` - 实时聊天消息

- **框架**：微信小程序原生开发

## 运行指南

### 1. MySQL数据库部署（新版本）

新版本已集成MySQL数据持久化，需要先设置数据库：

#### 1.1 环境配置
1. 复制环境变量文件：`cp .env.example .env`
2. 编辑`.env`文件，配置MySQL连接信息：
   ```env
    DB_HOST=127.0.0.1
    DB_PORT=3307
    DB_USER=avalon_user
    DB_PASS=<your_secure_password_here>
    DB_NAME=avalon_db
   ```

#### 1.2 初始化数据库
```bash
# 进入后端目录
cd server

# 安装依赖
npm install

# 初始化数据库（需要MySQL服务运行）
npm run db:init
# 或指定自定义.env文件路径: npm run db:init -- --env /path/to/.env
```

#### 1.3 数据库管理
- 数据库初始化脚本：`server/scripts/init-database.js`
- 数据库DDL文件：`mysql/DDL.sql`
- 数据库配置：`server/config/db.js`

### 2. 后端服务部署

```bash
# 进入后端目录
cd server

# 安装依赖（如果尚未安装）
npm install

# 开发模式（使用 nodemon 热重载）
npm run dev

# 生产模式
npm start
```

**服务启动后：**
- HTTP API: `http://localhost:8082` 或 `https://haoyu-wang141.top:8082`
- WebSocket: `ws://localhost:8082` 或 `wss://haoyu-wang141.top:8082`
- 健康检查: `GET /api/health`

### 2. 前端运行

1. **安装微信开发者工具**
2. **导入项目**：选择 `miniprogram` 目录
3. **配置 AppID**（测试号或个人 AppID）
4. **修改 API 配置**（如需本地测试）：
   ```javascript
   // miniprogram/services/api.js:3
   const BASE_URL = 'http://localhost:8082/api'; // 本地测试
   // const BASE_URL = 'https://haoyu-wang141.top:8082/api'; // 生产
   ```
5. **编译运行**：点击"编译"按钮

### 3. 测试流程

1. **启动后端**：确保服务器在 `8082` 端口运行
2. **打开前端**：在微信开发者工具中预览
3. **创建房间**：测试房间创建、加入、座位选择
4. **开始游戏**：至少5人准备后开始游戏
5. **游戏流程**：测试提名、投票、任务执行
6. **实时通信**：验证聊天功能和状态同步

## 开发指南

### 添加新页面（前端）

1. 在 `miniprogram/pages/` 目录下创建新目录，如 `pages/newpage/`
2. 创建四个文件：`.js`, `.json`, `.wxml`, `.wxss`
3. 在 `app.json` 的 `pages` 数组中添加路径

### 添加新API端点（后端）

1. 在 `server/routes/` 目录下创建新路由文件，或扩展现有路由
2. 在路由文件中添加新的端点处理逻辑
3. 在 `server/index.js` 中注册路由（如为新文件）
4. 在 `miniprogram/services/api.js` 中添加对应的API方法
5. 重启后端服务使更改生效

## 注意事项

1. **后端部署**：如需部署到公网（如 `haoyu-wang141.top:8082`），需要配置SSL证书和域名解析
2. **前端配置**：微信小程序需配置合法域名（`request` 和 `socket` 域名）
3. **数据持久化**：当前使用内存存储（Map对象），重启服务器数据丢失
4. **性能优化**：前端使用轮询（2秒间隔），可考虑改为WebSocket实时更新
5. **微信审核**：游戏类小程序需要特定资质，确保正确选择类目
6. **网络处理**：实现健壮的错误处理以应对多人游戏中的网络中断
